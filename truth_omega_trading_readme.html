Scopo

Questa demo mostra come applicare la metrica TruthΩ (stabilità/coerenza base-invariante) a dati finanziari, trasformando le variazioni giorno-su-giorno in un segnale sintetico con due derivati operativi:

  • Co⁺ — compressione esponenziale in [0,1] per normalizzare la stabilità
  • Score⁺ — punteggio dimostrativo per gating decisionale (segnale/rumore)

Nota: la formula qui è una riduzione didattica e non sostituisce la versione completa del Truth Engine.

Formato input richiesto (CSV)

Header minimo obbligatorio: date,open,high,low,close,volume. Esempio:

date,open,high,low,close,volume
2025-09-01,102,104,101,103,142000
2025-09-02,103,104,102,102.5,128000
...

La demo usa date, open, close. Le altre colonne possono essere ignorate.

Definizioni metriche (versione demo)

  • Δ = close - open
  • TruthΩ = sign(Δ) · ln(1 + |Δ|)
  • Co⁺ = exp(-|TruthΩ|)  ∈ (0,1]
  • Score⁺ = (Co⁺ · 100) − 10 · |TruthΩ|  (punteggio dimostrativo)
Intuizione TruthΩ
Stima “stabile” se piccoli Δ; cresce in modulo con movimenti più energici.
Co⁺
Schaccia l’ampiezza in [0,1]: facile da usare come peso/affidabilità.
Score⁺
Segnale unico per sogliatura semplice (demo).

Uso in browser

  1. Apri truth_omega_trading.html.
  2. Carica un file CSV (es. example_data.csv).
  3. Clicca Calcola → vedi tabella e grafico (TruthΩ / Score⁺).
  4. Opzionale: clicca Scarica risultati CSV per salvare l’output.

Uso via CLI (Python)

  1. Scarica truth_omega_trading.py e example_data.csv.
  2. Esegui: 
    python truth_omega_trading.py --input example_data.csv --output results_scored.csv
  3. Apri results_scored.csv e verifica colonne: date,TruthΩ,Co⁺,Score⁺.

Interpretazione rapida

  • TruthΩ ≈ 0 → giornata “neutra”/coerente (basso segnale direzionale).
  • |TruthΩ| ↑ → maggiore energia/asimmetria intraday.
  • Co⁺ vicino a 1 → elevata stabilità; vicino a 0 → instabilità.
  • Score⁺ → usa una soglia empirica (es. > 50) come filtro dimostrativo.

Le soglie dipendono dallo strumento/volatilità e vanno tarate su dati storici.

Limiti della demo

  • Usa solo open e close (no microstruttura, no volatilità realizzata).
  • Non esegue risk management, slippage, costi o sizing.
  • La formula è didattica; per ricerca/produzione usare il Truth Engine completo.
