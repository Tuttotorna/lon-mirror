# Public Navigation

This document gives a direct path for first-time visitors.

The visitor should not be forced to understand the whole ecosystem before seeing value.

---

## Start by need

| Need | Start here | Why |
|---|---|---|
| Understand the ecosystem | [lon-mirror](https://github.com/Tuttotorna/lon-mirror) | Public map and onboarding route |
| See proof first | [OMNIA-VALIDATION](https://github.com/Tuttotorna/OMNIA-VALIDATION) | Tests, artifacts, validation contracts |
| Understand the engine | [OMNIA](https://github.com/Tuttotorna/OMNIA) | Core structural measurement |
| Understand representation | [OMNIABASE](https://github.com/Tuttotorna/OMNIABASE) | Multi-representation / base-shift foundation |
| Detect signals before measuring | [OMNIA-RADAR](https://github.com/Tuttotorna/OMNIA-RADAR) | Candidate trace scanning |
| Check survival under transformation | [OMNIA-INVARIANCE](https://github.com/Tuttotorna/OMNIA-INVARIANCE) | Stability / collapse under controlled change |
| Decide when measurement should stop | [omnia-limit](https://github.com/Tuttotorna/omnia-limit) | Stop / continue boundary |
| Test stable-region candidates | [OMNIA-CONSTANT](https://github.com/Tuttotorna/OMNIA-CONSTANT) | Falsification pressure on candidate invariants |
| Orchestrate structural workflows | [OMNIAMIND](https://github.com/Tuttotorna/OMNIAMIND) | Workflow organization, not consciousness |
| Inspect chaotic divergence | [OMNIA-THREE-BODY](https://github.com/Tuttotorna/OMNIA-THREE-BODY) | Dynamic stress test |
| Inspect security-relevant traces | [OMNIA-SECURITY](https://github.com/Tuttotorna/OMNIA-SECURITY) | Bounded diagnostics, not scanner/verdict |
| Inspect crypto-like traces | [OMNIA-CRYPTO](https://github.com/Tuttotorna/OMNIA-CRYPTO) | Bounded diagnostics, not key recovery/proof of break |
