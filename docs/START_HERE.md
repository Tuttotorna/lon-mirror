# Start Here

This is the shortest entrypoint into the MB-X.01 / L.O.N. ecosystem.

## One sentence

MB-X.01 / L.O.N. is an ecosystem for measuring structural behavior under transformation while keeping measurement, inference, and decision separate.

```text
measurement != inference != decision
```

## The core idea

A system can sound coherent while being structurally fragile.

A representation can look stable while depending on an arbitrary frame.

A process can continue producing output after no new structure remains.

The ecosystem exists to expose these conditions structurally.

## Where to go first

Read in this order:

1. `README.md`
2. `ECOSYSTEM.md`
3. `docs/MASTER_MAP.md`
4. `docs/BOUNDARIES.md`

Then go to the specific repository:

- OMNIA if you want the measurement core
- OMNIABASE if you want the representation / base-invariance layer
- OMNIAMIND if you want orchestration around reasoning
- omnia-limit if you want the terminal boundary layer
- OMNIA-VALIDATION if you want evidence, failures, and reproducible runs
- OMNIA-INVARIANCE if you want transformation/invariance testing

## Minimal architecture

```text
representation -> measurement -> orchestration/gate -> boundary -> external decision
```

## What not to assume

Do not read the ecosystem as:

- artificial consciousness
- semantic truth engine
- autonomous decision system
- final proof system
- replacement for expert judgment

The safe reading is:

```text
observe structure
measure stability
detect fragility
record limits
keep decision external
```
